Statebus

Michael Toomim and the Invisible College Anyone can add a website to the open web, but the insides of websites are closed. Statebus extends the web protocol to the insides of websites, by giving each piece of internal state a URL, and making servers promise to update their clients automatically when state changes. These protocol changes enable a new programming architecture that simplifies programming with distributed state and unifies client and server. This makes it both easier to develop the internals of a website, while simultaneously making it easier to share the internals with other websites. Statebus makes three extensions to HTTP:
  1. Each piece of state has a URL and a standard format, not just every page.
  2. A piece of state can contain other linked pieces of state, that are fetched along with it
  3. Each state is automatically promised to be updated when changed, not just transferred.
